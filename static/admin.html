添加模型映射

在 Cursor 自定义模型中添加的名称
发送到中转站的真实模型名称
anthropic：转换为 Anthropic Messages 格式 — 适用于中转站通过 /v1/messages 提供 Claude 模型
openai：保持 OpenAI Chat Completions 格式 — 适用于 GPT、DeepSeek、Codex 或通过 /v1/chat/completions 提供所有模型的中转站
responses：保持 OpenAI Responses 格式 — 适用于中转站仅通过 /v1/responses 提供模型能力
gemini：转换为 Gemini Contents 格式 — 适用于 Google Gemini API 或兼容的中转站
自动检测：根据上游模型名判断（含 claude → anthropic，含 gemini → gemini，其他 → openai）
每个模型可单独配置。指令会被注入到发往上游的请求中，模型会无条件信任该内容。
适用场景：自定义人格、解除限制、注入 CTF 环境假设等。
前置：自定义指令放在系统提示词最前面，模型优先看到；后置：放在末尾
对转发到上游的请求体做字段级增删改。值为 null 表示删除该字段，其余为设置/覆盖。
适用场景：注入 reasoning_effort、删除上游不支持的 stream_options 等。
对转发到上游的请求头做增删改，规则同 Body 修改。
适用场景：按模型设置不同的 Authorization、API 版本头等。
